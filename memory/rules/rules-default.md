# Regras Padrao

## Codigo
- Seguir padrao dos modulos existentes no projeto
- Tipos/Interfaces para dados publicos
- Funcoes com assinatura clara
- Comentarios apenas quando a logica nao e obvia

## Testes
- Todo modulo deve ter testes
- Framework do projeto (pytest, vitest, jest, etc.)
- Cobrir: caso feliz, edge cases, erros esperados

## Seguranca
- Nao exfiltrar dados privados
- Nao commitar chaves ou tokens
- Preferir trash/backup sobre delete permanente
