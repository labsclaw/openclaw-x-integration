# Paperclip CEO Agent Fix — Protocol Mismatch Resolved

## Problema
CEO agent (OpenClaw) no Paperclip em loop infinito de falha: `protocol mismatch` ao conectar ao OpenClaw gateway (`ws://127.0.0.1:18789/`).

## Causa Raiz (dupla)
1. **PROTOCOL_VERSION = 3** no adapter `@paperclipai/adapter-openclaw-gateway`, mas OpenClaw gateway v2026.5.19 exige v4
2. **Propriedade `paperclip`** no payload do `agentParams` — o gateway rejeita props desconhecidas no JSON Schema

## Fixes aplicados (no cache do npx)
Arquivo: `$env:LOCALAPPDATA\npm-cache\_npx\43414d9b790239bb\node_modules\@paperclipai\adapter-openclaw-gateway\dist\server\execute.js`

1. `const PROTOCOL_VERSION = 3` → `const PROTOCOL_VERSION = 4` (linha 4)
2. Substituído `agentParams.paperclip = paperclipPayload` por filtro no spread do `payloadTemplate` para remover a propriedade `paperclip`

## Fonte
PR #6480 do RyanTJoy no GitHub paperclipai/paperclip
- "bump PROTOCOL_VERSION to 4 and strip agentParams.paperclip"
- Já mergeado como hotfix no source; só não tinha sido publicado no npm ainda

## Comandos
- Iniciar Paperclip: `npx paperclipai run`
- Script de startup: `~\AppData\Local\paperclip\start-paperclip.ps1`
